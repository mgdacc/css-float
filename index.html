Float

¿Qué es Float?

Originalmente, float se creó para imitar el estilo de los periódicos: una imagen a un lado y el texto fluyendo alrededor de ella.

img {
  float: left;
  margin-right: 20px;
}

/* La imagen se pega a la izquierda, 
   el texto sube y la rodea. */
2 / 14

Valores de Float

Es una propiedad simple con pocos valores posibles.

.elemento {
  float: left;   /* Flota a la izquierda */
  float: right;  /* Flota a la derecha */
  float: none;   /* No flota (defecto) */
  float: inherit;
}
3 / 14

Comportamiento del Flujo

Cuando un elemento flota:

  • Es removido del flujo normal del documento.
  • Se mueve hacia la izquierda o derecha hasta tocar el borde de su contenedor.
  • El contenido (texto) de los elementos adyacentes respeta su espacio y lo rodea.
4 / 14

El Pasado Oscuro: Layouts

Antes de Flexbox y Grid (2005-2015), usábamos float para crear columnas. Era difícil y propenso a errores.

.columna-1 {
  float: left;
  width: 30%;
}
.columna-2 {
  float: left;
  width: 70%;
}
/* ¡No hagas esto hoy! Usa display: flex. */
5 / 14

Propiedad Clear

Especifica que un elemento no debe tener elementos flotantes a su lado. Es el freno de mano del float.

footer {
  /* Baja el footer hasta que no haya nada 
     flotando a su izquierda o derecha. */
  clear: both; 
  
  /* Otros valores: left, right, none */
}
6 / 14

El Problema: Colapso del Padre

Si un contenedor tiene solo hijos flotantes, su altura colapsa a 0 píxeles. El padre "no siente" a los hijos porque están flotando.

.padre {
  border: 1px solid red;
  /* Altura resultante: 0px. El borde se ve como una línea.
     Los hijos se salen del contenedor. */
}
.hijo { float: left; }
7 / 14

Solución Antigua: Div Vacío

La vieja escuela solucionaba esto añadiendo HTML basura al final del contenedor.

<div class="padre">
   <div class="hijo">...</div>
   <div class="hijo">...</div>
   
   <!-- Elemento "basura" solo para limpiar -->
   <div style="clear: both;"></div>
</div>
8 / 14

Solución Clásica: El "Clearfix"

Un hack de CSS usando pseudo-elementos para no ensuciar el HTML. Fue el estándar durante años.

.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

/* Se aplica la clase .clearfix al elemento padre */
9 / 14

Solución Moderna 1: Overflow

Establecer el overflow distinto de visible fuerza al padre a reconocer la altura de los hijos flotantes (crea un BFC).

.padre {
  overflow: hidden; 
  /* O overflow: auto */
}

/* Desventaja: Puede cortar contenido que 
   quieras que sobresalga (como sombras o menús). */
10 / 14

Solución Moderna 2: Flow-Root

La forma oficial y moderna de contener floats sin efectos secundarios.

.padre {
  display: flow-root;
}

/* Crea un nuevo Block Formatting Context (BFC).
   El padre abraza a los hijos flotantes perfectamente. */
11 / 14

¿Cuándo usar Float hoy?

Vuelve a sus raíces: Úsalo para diseño editorial, imágenes dentro de artículos y Letras Capitales.

article img {
  float: right;
  margin-left: 20px;
  width: 200px;
}
/* El texto fluirá alrededor de la imagen. 
   Flexbox no puede hacer esto fácilmente. */
12 / 14

¿Cuándo NO usar Float?

No lo uses para la estructura de la página (Layout).

// MALA IDEA:
.sidebar { float: left; width: 30%; }
.main { float: right; width: 70%; }

// BUENA IDEA:
.contenedor { display: grid; grid-template-columns: 30% 70%; }
// O
.contenedor { display: flex; }
13 / 14

Resumen

Float es una herramienta de nicho hoy en día.

1. `float` saca al elemento del flujo normal.
2. Texto y contenido inline rodean al elemento flotado.
3. Los padres colapsan si sus hijos flotan.
4. Usa `display: flow-root` en el padre para arreglarlo.
5. Usa Grid/Flex para columnas, `float` solo para texto/img.
